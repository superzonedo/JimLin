# GitHub 協作指南

## 🎯 快速分享方式

### 方式 1：直接分享 GitHub 連結（最簡單）

**您的倉庫地址**：
```
https://github.com/superzonedo/JimLin
```

直接將這個連結分享給其他人即可！

## 👥 協作方式

### 方式 1：添加協作者（適合團隊內部協作）

**步驟**：

1. **前往倉庫設置頁面**：
   - 點擊 GitHub 倉庫頁面右上角的 "Settings"
   - 或直接訪問：`https://github.com/superzonedo/JimLin/settings`

2. **添加協作者**：
   - 在左側選單中找到 "Collaborators"（協作者）
   - 點擊 "Add people" 按鈕
   - 輸入對方的 GitHub 用戶名或電子郵件
   - 選擇權限級別：
     - **Read**：只能查看和克隆
     - **Write**：可以推送代碼（推薦）
     - **Admin**：完全管理權限

3. **對方會收到邀請郵件**：
   - 對方接受邀請後，就可以直接推送代碼了

**優點**：
- ✅ 簡單直接
- ✅ 適合小團隊
- ✅ 可以直接推送，不需要 Fork

### 方式 2：Fork + Pull Request（適合開源項目或外部貢獻者）

**流程**：

1. **對方 Fork 您的倉庫**：
   - 對方在 GitHub 上點擊 "Fork" 按鈕
   - 會創建一個他們自己的副本

2. **對方克隆並修改**：
   ```bash
   git clone https://github.com/對方的用戶名/JimLin.git
   cd JimLin
   # 進行修改...
   git add .
   git commit -m "修改說明"
   git push
   ```

3. **對方創建 Pull Request**：
   - 在對方的 Fork 頁面點擊 "Contribute" → "Open pull request"
   - 填寫修改說明
   - 您會收到通知，可以審核並合併

**優點**：
- ✅ 適合開源項目
- ✅ 您可以審核所有更改
- ✅ 不會直接修改主倉庫

## 🔐 權限設置

### 查看當前協作者

1. 前往：`https://github.com/superzonedo/JimLin/settings/access`
2. 查看 "Collaborators" 區塊

### 權限級別說明

| 權限 | 可以做的事 | 適合對象 |
|------|-----------|---------|
| **Read** | 查看、克隆、下載 | 只讀用戶 |
| **Write** | 以上 + 推送代碼、創建分支 | 開發團隊成員（推薦） |
| **Admin** | 以上 + 管理設置、刪除倉庫 | 項目管理員 |

## 📋 協作工作流程

### 標準 Git 工作流程

1. **克隆倉庫**：
   ```bash
   git clone https://github.com/superzonedo/JimLin.git
   cd JimLin
   ```

2. **創建功能分支**（推薦）：
   ```bash
   git checkout -b feature/新功能名稱
   # 或
   git checkout -b fix/修復問題
   ```

3. **進行修改並提交**：
   ```bash
   # 修改文件...
   git add .
   git commit -m "描述您的修改"
   ```

4. **推送到 GitHub**：
   ```bash
   git push origin feature/新功能名稱
   ```

5. **創建 Pull Request**（如果使用分支）：
   - 在 GitHub 上會自動提示創建 PR
   - 或前往：`https://github.com/superzonedo/JimLin/pulls`
   - 點擊 "New pull request"

6. **審核和合併**：
   - 您或其他協作者審核代碼
   - 通過後點擊 "Merge pull request"

### 直接推送到 main 分支（簡單項目）

如果項目較小，也可以直接推送到 main：

```bash
git add .
git commit -m "修改說明"
git push origin main
```

⚠️ **注意**：多人協作時建議使用分支，避免衝突

## 🛠️ 解決衝突

如果多人同時修改同一文件，可能會出現衝突：

1. **拉取最新代碼**：
   ```bash
   git pull origin main
   ```

2. **解決衝突**：
   - Git 會標記衝突的地方
   - 手動編輯文件解決衝突
   - 刪除衝突標記（`<<<<<<<`, `=======`, `>>>>>>>`）

3. **提交解決方案**：
   ```bash
   git add .
   git commit -m "解決衝突"
   git push origin main
   ```

## 📝 最佳實踐

### 1. 使用有意義的提交訊息

```bash
# ❌ 不好的提交訊息
git commit -m "修改"

# ✅ 好的提交訊息
git commit -m "修復 Google 登入錯誤"
git commit -m "添加用戶個人資料頁面"
git commit -m "優化圖片上傳性能"
```

### 2. 頻繁提交和推送

- 完成一個功能就提交一次
- 每天結束前推送代碼
- 避免大量修改一次性提交

### 3. 使用分支進行功能開發

```bash
# 創建功能分支
git checkout -b feature/user-profile

# 開發完成後
git push origin feature/user-profile

# 創建 Pull Request 合併到 main
```

### 4. 定期同步主分支

```bash
# 切換到 main 分支
git checkout main

# 拉取最新代碼
git pull origin main

# 切換回功能分支
git checkout feature/your-feature

# 合併 main 的更新
git merge main
```

## 🆘 常見問題

### Q: 如何撤銷錯誤的提交？

```bash
# 撤銷最後一次提交（保留修改）
git reset --soft HEAD~1

# 完全撤銷最後一次提交（刪除修改）
git reset --hard HEAD~1
```

### Q: 如何查看誰修改了什麼？

```bash
# 查看文件修改歷史
git log --follow 文件名

# 查看誰修改了哪一行
git blame 文件名
```

### Q: 如何恢復到之前的版本？

```bash
# 查看提交歷史
git log

# 恢復到特定提交
git checkout 提交hash

# 或創建新分支從該提交開始
git checkout -b 新分支名 提交hash
```
